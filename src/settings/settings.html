Swift Selection Search (SSS) options page

Instructions
Quick setup!
  1. Add new search engines using the "Add engine" button in the "Search engines" section. You can also load your search bar engines from Firefox. Engines will appear in the SSS popup and context menu when you select text on a page. SSS comes with a few examples!
  2. You can change the engine names, search URLs, and icon URLs. When you click a search engine in the popup or context menu, {searchTerms} (in the search URL) will be replaced by the selected text.
    [Press to see advanced usage]
    • If {hostname} is present, it will be replaced by the website domain you are on.
    • {searchTerms} supports text substitutions, in the format {searchTerms{OriginalText|TargetText}}, supporting multiple pairs of {OriginalText|TargetText}. So if in a certain search engine you want to always replace spaces in your search with "+" and "@" with "at" you can use {searchTerms{ |+}{@|at}}
    • Since SSS uses the special characters \, {, |, or } for delimiting the substitutions, if you want to use these characters in the substitution itself, you should precede them with a \. For example, {searchTerms{\{|\}}} replaces { with } in your text and {searchTerms{ |\\} replaces spaces with a \.
  3. Having trouble finding icons? Maybe try this favicon finder or this one. :)
  4. For customizing the popup look, context menu, click behaviour, etc, please check the other options in this page.
Troubleshooting
Having problems?
  • If an icon does not appear and its URL is correct, please check if that still happens if you disable your ad blocker and add the icon URL again. Some blocking lists in your ad blocker can stop certain images from downloading! (ex.: YouTube may be blocked by "Fanboy's Annoyance List")
  • Some search engine icons may not appear if they are "resource://" links (some imported engines have those). This is normal, unfortunately. Please try finding another icon for the engine here!
  • Does the popup not appear at all on some websites or frames of a website? It may be a website content policy or an incompatibility with SSS. For example, addons.mozilla.org disables page modifications, and bugzilla.mozilla.org blocks most images. If you find another site, please report it using the issues link below and I'll see what I can do!
  • Is your problem not here? Or maybe you found a bug? Contact me or post an issue here.

Popup/icons behaviour
Opening behaviour
When to open the popup.
(default: Auto)
Middle mouse click margin
Adds a margin in pixels so you can middle-click near the selection as well as on it.
(default: 14)
Popup location
Where the popup should appear after selecting text.
(default: At cursor location)
Left mouse button click
What to do when a search engine is clicked with the left mouse button.
(default: Open in this tab)
Middle mouse button click
What to do when a search engine is clicked with the middle mouse button.
(default: Open in new background tab)
Minimum selected characters
In "Auto" opening behaviour, only shows popup if you have at least this number of characters selected (discarding surrounding spaces). Setting to 0 always shows.
(default: 0)
Allow popup on editable fields
Show the popup when text is selected with the mouse on an editable text field, such as a login form, a search bar, or an email composing window. NOTE: "Shortcut to open popup" ignores this setting and always works on editable fields.
(default: Off)
Hide popup on page scroll
Hides the popup when the page moves.
(default: On)
Hide popup on right click
Hides the popup when the right mouse button is pressed outside the popup. NOTE: Left or middle clicks always hide it regardless.
(default: On)
Hide popup on search engine click
Hides the popup when a search engine is clicked.
(default: On)
Shortcut to open popup
Ctrl+Shift+Space Cmd+Shift+Space
[NOTE: Only configurable in Firefox 60+]
 Opens the popup if you have text selected. Works for editable fields even if "Allow popup on editable fields" is Off. Useful if "Opening behaviour" is set to Keyboard-only.
[Press for details on how to write the shortcut] Use Ctrl (it means Command on Mac), Alt, MacCtrl (actual Control key on Mac), Shift and the desired key. Check this link for more details.
NOTE: Many shortcuts are reserved for the browser. If yours doesn't work please try another combination!
(default: Ctrl+Shift+Space)
Shortcut to toggle auto popup opening
Ctrl+Shift+U Cmd+Shift+U
[NOTE: Only configurable in Firefox 60+]
 Toggles "Opening behaviour" setting between Auto and Keyboard-only modes (if not Off). Use this when the popup is getting in your way.
[Press for details on how to write the shortcut] Use Ctrl (it means Command on Mac), Alt, MacCtrl (actual Control key on Mac), Shift and the desired key. Check this link for more details.
NOTE: Many shortcuts are reserved for the browser. If yours doesn't work please try another combination!
(default: Ctrl+Shift+U)

Storage & backup
Storage size
This is an estimate size of your SSS engines and settings. Engines with "data:image" icons tend to occupy a lot of space. WARNING: Actual size will very likely be above or below the displayed value.
Sync settings (upload)
Saves all SSS settings and engines in Firefox Sync. This does not force upload, so you may have to wait for Firefox to sync or force the sync yourself from Firefox's menu. Beware that storage size should be below 100KB for this to work. Estimate size is not exact, so it may still work if above the limit, or fail if below it. If above the limit, start by deleting any "data:image" URLs in your engines, or any other extremely long URLs.
Sync settings (download)
Downloads previously saved SSS settings and engines from Firefox Sync. This will override all existing settings and engines!
Export settings to file
Exports all SSS settings and engines to a file.
Import settings from file
Imports SSS settings and engines from a file (previously exported). This will override all existing settings and engines!
Reset
Reset settings
Resets all options except search engines to their defaults (as if running SSS for the first time).
